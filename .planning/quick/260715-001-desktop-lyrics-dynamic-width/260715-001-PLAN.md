# Quick Task 260715-001: 桌面歌词窗口动态宽度

## Goal
桌面歌词窗口宽度根据歌词文本长度动态调整（既能放大也能缩小），不超过最大宽度限制。

## Background
当前 `internal/desktop_lyrics/desktop_lyrics_darwin.go` 中：
- `minWinW = fontSize * 20` 过大，导致短歌词留下大量空白
- `updateScrollNeed()` 只放大窗口，不缩小
- 底部标签 (posSecond) 从左侧锚定，窗口缩小时位置不正确

## Tasks

### Task 1: 减小最小窗口宽度
- **File:** `internal/desktop_lyrics/desktop_lyrics_darwin.go`
- **Action:** 将 `minWinW = fontSize * 20` 改为 `minWinW = fontSize * 5`（足够显示几行字符的最小宽度）
- **Verify:** minWinW 足够容纳约5个中文字符

### Task 2: add shrink logic in updateScrollNeed
- **File:** `internal/desktop_lyrics/desktop_lyrics_darwin.go`
- **Action:** 在 `updateScrollNeed()` 中，当 `neededWinW < c.currentWinW` 时，缩小窗口到 `max(neededWinW, c.minWinW)`
- **Verify:** 短歌词时窗口自动缩小，长歌词时窗口自动放大

### Task 3: 修复 posSecond（底部标签）右锚定
- **File:** `internal/desktop_lyrics/desktop_lyrics_darwin.go`
- **Action:** `resizeWindow()` 中 posSecond 的 x origin 改为 `newW - padding - textW`，使底部标签从右侧锚定
- **Verify:** 窗口缩小时底部右对齐文本仍然紧贴右侧
