贡献指南

让 go-musicfox 变得更好——每个贡献都值得被感谢。

go-musicfox 是一个 MIT 开源项目，欢迎任何形式的贡献。无论是提交代码、报告 Bug、改进文档、还是参与社区讨论，你的参与都会让这个项目变得更好。

01 行为准则

在参与之前，请阅读并遵守我们的 行为准则。我们致力于为每个人创造友好、尊重、包容的协作环境。

02 寻找贡献方向

不知道怎么开始？以下是几个常见的贡献方向：

  • 报告 Bug — 在 GitHub Issues 搜索是否已有相同报告，如果没有则创建新 Issue
  • 提交功能请求 — 在 Discussions 中提出你的想法，获得维护者反馈后再实现
  • 改进文档 — README、Wiki、注释或官网内容
  • 完善测试 — 增加单元测试、集成测试覆盖
  • 代码贡献 — 查看标记了 good first issue 或 help wanted 的 Issue

03 开发环境搭建

确保已安装 Go 1.22+，然后克隆仓库：

git clone https://github.com/go-musicfox/go-musicfox.git
cd go-musicfox
go mod download
make          # 编译到 bin/
make install  # 安装到 $GOPATH/bin

04 提交代码

我们遵循 Feature Branch 工作流和 Conventional Commits 规范：

git checkout -b feat/your-feature
git commit -m "feat(xxx): 简短的描述"
git push origin feat/your-feature

然后在 GitHub 上创建 Pull Request，关联相关的 Issue 编号。

提交类型速查

feat新功能
fixBug 修复
docs文档更新
refactor代码重构
test测试相关

05 Pull Request 流程

  1. 确保 PR 描述清晰，说明改动原因和实现方式
  2. 关联相关 Issue（如 Closes #123）
  3. 确保 CI 检查通过
  4. 至少一位维护者 Review 后合并

06 其他贡献方式

  • Star 项目 — 在 GitHub 上点亮 Star，帮助项目被更多人发现
  • 分享推荐 — 在博客、社交媒体分享你的使用体验
  • 反馈问题 — 使用中遇到任何问题，在 Issues 中告诉我们
  • 参与讨论 — 在 Discussions 中帮助其他用户
  • 打包分发 — 为更多包管理器（Homebrew、AUR、Nix 等）维护打包文件

感谢你花时间贡献 go-musicfox ❤️
